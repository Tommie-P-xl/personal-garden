# 个人网站设计文档

> **项目类型**：Digital Garden 风格个人网站
> **创建日期**：2025-06-25
> **状态**：已批准

---

## 1. 项目概述

### 1.1 目标

构建一个 Digital Garden（数字花园）风格的个人网站，用于：

- 展示个人简历和项目经历（求职用途）
- 记录和整理学习笔记（Linux 内核、编程等）
- 存储其他领域笔记（家教备课、自媒体技巧等）
- 支持多媒体内容（文字、图片、视频）

### 1.2 核心价值

- **知识管理**：笔记像花园一样持续生长，不同领域通过标签和链接自然串联
- **求职展示**：向面试官展示体系化学习能力和知识管理能力
- **高效复习**：快速查阅曾经学过的知识，支持增量更新

## 3. 页面设计

### 3.1 首页（Home / 简历）

- 个人信息卡片：头像、姓名、一句话介绍、联系方式
- 教育背景
- 实习/工作经历
- 技能栈（可视化进度条）
- 荣誉/证书

### 3.2 笔记花园（Garden）

- 分类筛选栏：全部 / Linux / 家教 / 自媒体 / 编程
- 笔记卡片网格：显示标题、分类、日期、生长状态
- 笔记详情页：正文 + 双向链接 + 关系图谱 + 相关笔记

### 3.3 项目展示（Projects）

- 项目卡片：名称、描述、技术栈标签、链接
- 项目详情页：完整介绍、截图、技术细节

### 3.4 关于我（About）

- 个人故事 / 学习历程
- 兴趣爱好
- 联系方式汇总
- 本站技术说明

## 4. Digital Garden 核心功能

### 4.1 笔记生长状态

在笔记 frontmatter 中设置 `growth` 字段：

| 状态 | 标记 | 含义 |
|------|------|------|
| 🌱 种子 | `growth: seed` | 刚开始记录，几句话/几个要点 |
| 🌿 发芽 | `growth: sprout` | 内容基本成型，有结构但不完整 |
| 🌳 成熟 | `growth: evergreen` | 内容完整，可作为参考资料 |

### 4.2 双向链接

使用 `[[笔记名]]` 语法在笔记中引用其他笔记：

```markdown
进程调度是 Linux 内核的核心机制之一，与 [[内存管理]] 密切相关。
```

- 自动生成反向链接（被引用列表）
- 笔记底部显示"相关笔记"

### 4.3 关系图谱

- 使用 D3.js 或类似库渲染交互式知识图谱
- 节点代表笔记，边代表链接关系
- 点击节点可跳转到对应笔记

### 4.4 全文搜索

- 使用 Pagefind 实现客户端搜索
- 支持搜索标题、内容、标签
- 无需后端服务

### 4.5 标签系统

- 标签云展示
- 按标签筛选笔记
- 标签颜色分类

## 5. 笔记格式规范

### 5.1 Frontmatter

```yaml
---
title: Linux 进程调度机制
category: linux
tags: [内核, 进程, 调度]
growth: sprout
created: 2025-06-25
updated: 2025-06-25
summary: 学习 Linux 进程调度的核心概念
---
```

### 5.2 正文格式

- 支持标准 Markdown 语法
- 支持图片：`![描述](/images/xxx.png)`
- 支持视频嵌入
- 支持双向链接：`[[笔记名]]`
- 支持 MDX 组件嵌入

## 6. 部署流程

```
写笔记 (VS Code / Obsidian)
    ↓
Git Push (推送到 GitHub)
    ↓
GitHub Actions (自动构建)
    ↓
GitHub Pages (网站更新)
```

- 每次 `git push` 自动触发构建和部署
- 无需手动操作
- 支持自定义域名（可选）

## 7. 视觉设计

### 7.1 风格

- Digital Garden 风格：极简、留白多、护眼配色
- 米白/浅灰底色，确保文字清晰可见
- 深色文字，保证良好的对比度

### 7.2 配色方案

- 背景：`#fafafa`（浅灰白）
- 主色调：`#2d5016`（深绿，花园主题）
- 强调色：`#4a7c2e`（中绿）
- 文字：`#1a1a1a`（近黑）
- 次要文字：`#555555`（深灰）

### 7.3 响应式设计

- 移动端优先
- 支持手机、平板、桌面三种断点
