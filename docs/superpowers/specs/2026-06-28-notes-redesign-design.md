# 笔记系统重构设计文档

## 概述

将现有笔记系统重构为菜鸟教程风格的教程式布局，保留数字花园的双向链接、生长状态等特性。核心变化：笔记按目录层级组织，详情页采用左侧目录导航 + 右侧教程内容的布局。

## 目标

- 笔记详情页采用教程式布局（左侧目录树 + 右侧内容）
- 支持最多三级子目录的笔记组织
- 序号前缀控制排序，`_meta.json` 定义显示名称
- 支持 LaTeX 数学公式（KaTeX）
- 代码块支持暗黑/亮色双主题 + 一键复制
- 保留花园特性：双向链接、标签、反向链接
- 简化 frontmatter，分类由目录结构决定
- 添加新笔记只需创建文件 + git push

## 笔记文件结构

```
src/content/notes/
├── 0001-linux/                     # 一级分类（带序号）
│   ├── _meta.json                  # {"title": "Linux", "icon": "🐧"}
│   ├── 0001-memory-management/     # 二级主题（带序号）
│   │   ├── _meta.json              # {"title": "内存管理", "icon": "🧠"}
│   │   ├── 0001-虚拟内存.md        # 笔记
│   │   ├── 0002-分页机制.md
│   │   ├── 0003-伙伴系统.md
│   │   └── 0004-advanced/          # 三级子目录（带序号）
│   │       ├── _meta.json          # {"title": "进阶"}
│   │       ├── 0001-Slab分配器.md
│   │       └── 0002-页面置换.md
│   └── 0002-process-scheduling/
│       ├── _meta.json
│       ├── 0001-CFS调度器.md
│       └── 0002-进程状态.md
└── 0002-coding/
    ├── _meta.json
    └── 0001-python/
        ├── _meta.json
        ├── 0001-基础语法.md
        └── 0002-数据类型.md
```

### 命名约定

- **四位数序号前缀**：`0001-`、`0002-`、`0013-` 控制同级内的排序，笔记和目录都带序号
- **序号决定顺序**：同级目录下，笔记和子目录按序号混合排列，如 `0001-笔记.md`、`0002-子目录/`、`0003-笔记.md`
- **渲染时去掉序号**：`0001-虚拟内存.md` 显示为 `虚拟内存`，`0004-advanced/` 显示为 `_meta.json` 中的 `title`
- **`_meta.json`**：定义目录的显示名称、可选图标和描述
- **最多三级目录**：一级分类 → 二级主题 → 三级子主题
- **向后兼容**：现有顶层 md 文件（如 `linux/memory-management.md`）视为该目录下的单篇笔记

### `_meta.json` 格式

```json
{
  "title": "内存管理",
  "description": "Linux 内存管理相关笔记",
  "icon": "🧠"
}
```

**必填**：`title`（目录显示名称）
**可选**：`description`、`icon`
**默认行为**：如果目录下没有 `_meta.json`，使用目录名作为 `title`，无图标。

### frontmatter 格式（简化）

```yaml
---
title: 伙伴系统
summary: Linux 内核的内存分配算法
tags: [内核, 内存, 分配]
---
```

不再需要 `category`（由目录决定）、`growth`（简化掉）、`created`（由 git 历史决定）。

### 向后兼容策略

现有顶层 md 文件（如 `linux/memory-management.md`）的处理：
- 重构时将现有文件迁移到新目录结构中
- 例如：`linux/memory-management.md` → `linux/memory-management/01-基础概念.md`
- 迁移是单向的，重构完成后旧格式不再支持

## 页面布局

### 笔记列表页（/garden/）

保留现有卡片列表样式，作为笔记入口页。卡片展示：
- 标题（从 frontmatter 读取）
- 摘要（summary）
- 标签
- 分类（从一级目录的 `_meta.json` 推断）

**笔记发现机制**：构建时递归扫描 `src/content/notes/` 下所有 `.md` 文件（排除 `_meta.json`），每个 md 文件生成一个卡片。列表页按一级分类分组展示。

### 侧边栏组件

**特性**：
- 从文件系统动态生成目录树
- 笔记和子目录按四位数序号混合排列（`0001-笔记.md`、`0002-子目录/`、`0003-笔记.md`）
- 当前文件高亮（绿色背景 + 粗体）
- 目录组可折叠/展开（默认全部展开）
- 序号前缀在显示时去掉（`0001-虚拟内存.md` → `虚拟内存`，`0004-advanced/` → 显示 `_meta.json` 中的 `title`）
- 响应式：桌面端左侧固定（宽度 260px），移动端抽屉式弹出
- 暗黑模式完整支持

### 上/下篇导航

- 基于同目录下的序号排序确定上下篇
- 跨目录时链接到相邻目录的最后/第一篇
- 显示格式：`[← 上一篇: 标题]` `[下一篇: 标题 →]`

## 内容渲染

### 代码块

- **语法高亮**：Shiki（Astro 内置），双主题配置
  - 亮色：`github-light`
  - 暗色：`github-dark`
- **复制按钮**：右上角，点击后显示 ✓ 反馈
- **暗黑模式切换**：跟随全局主题自动切换代码配色

### LaTeX 数学公式

- **库**：KaTeX（轻量、快速）
- **集成方式**：Astro remark 插件（`remark-math` + `rehype-katex`）
- **语法**：
  - 行内：`$E=mc^2$`
  - 块级：`$$\sum_{i=1}^{n} x_i$$`

### 文章排版（菜鸟教程风格）

- h2 标题：底部有分割线（`<hr>`）
- 表格：斑马纹行、清晰边框、圆角
- 引用块：左侧绿色竖条背景
- 图片：圆角 + 阴影
- 双向链接：`[[笔记标题]]` 语法，渲染为带图标的链接

## 技术实现要点

### 路由

- 使用 `[...slug].astro` 捕获任意深度路径
- 构建时扫描 `src/content/notes/` 生成所有可能的路径
- `getStaticPaths()` 递归遍历目录树

### 侧边栏数据

- 构建时通过 `fs.readdirSync` 递归扫描目录
- 解析 `_meta.json` 获取显示名称
- 按文件名序号排序
- 将目录树数据作为 props 传递给页面组件

### 内容集合

- 保留现有 content collection 用于列表页
- 详情页改用 `import.meta.glob()` 或直接 `fs` 读取 md 文件
- 用 `astro:content` 的 `render()` 函数渲染 markdown

### KaTeX 集成

```bash
npm install remark-math rehype-katex katex
```

在 `astro.config.mjs` 中配置 markdown remark/rehype 插件。

### 代码块双主题

在 `astro.config.mjs` 的 `shikiConfig` 中配置双主题：

```js
shikiConfig: {
  themes: {
    light: 'github-light',
    dark: 'github-dark',
  }
}
```

## 添加新笔记的流程

```
1. 确定笔记属于哪个分类/主题目录
2. 创建文件：{四位数序号}-{标题}.md（如 0003-伙伴系统.md）
3. 写 frontmatter（title, summary, tags）
4. 写内容（支持 LaTeX、代码块、表格、双向链接）
5. git add → commit → push
6. GitHub Actions 自动构建部署
```

如果需要新建子目录：
1. 创建目录：`{四位数序号}-{名称}/`（如 `0004-进阶/`）
2. 在目录下创建 `_meta.json`（定义 title/icon）
3. 在目录下创建带序号的 md 文件
4. git push

## 与现有系统的关系

- **列表页**：保留现有卡片样式，数据源改为从新目录结构扫描
- **首页**：技术栈标签仍从笔记 tags 自动提取
- **项目页**：不受影响
- **关于页**：不受影响
- **双向链接**：保留 `[[笔记标题]]` 语法
- **暗黑模式**：所有新组件都支持
